    **Data Model Change — `buildOrgChart()` replaces `buildAgentTree()`:**

    The current `buildAgentTree()` produces `TreeDepartment[]` (flat list of departments, each with leads). The new function produces a single `OrgChartNode` tree:

    ```typescript

      id: string;
      name: string;
      role: string | null;
      status: string;
      type: "root" | "lead" | "sub-agent";
      departmentName: string | null;
      departmentType: string | null;
      departmentId: string | null;
      model_profile: Record<string, unknown>;
      skill_count: number;
      isCollapsed: boolean;
      children: OrgChartNode[];
    ```

    The `buildOrgChart()` function:
    1. Find the Owner department's lead agent — this becomes the ROOT node at the top of the tree.
       - If no Owner department or no lead in Owner, create a synthetic root: `{ id: "root", name: businessName ?? "Organization", role: "Root", type: "root", status: "active", departmentName: null, departmentType: null, departmentId: null, model_profile: {}, skill_count: 0, isCollapsed: false, children: [] }`
    2. For each OTHER department (Sales, Support, Operations, custom), find its lead agent(s). These become DIRECT CHILDREN of the root node.
    3. For each lead agent, find its sub-agents (agents where parent_agent_id = lead.id). These become children of the lead.
    4. Departments with no agents are NOT rendered as nodes (they have no agent to show). They can optionally be mentioned as empty placeholders, but NOT as full-width bars.
    5. If the Owner department has sub-agents under its lead, those are also children of the root node (alongside the other department leads).
    6. Result: a single OrgChartNode tree rooted at the Owner/business node.

    **Elbow/Step Connectors — Replace bezier curves in `AgentTreeLines`:**

    Rewrite `agent-tree-lines.tsx` to draw elbow/step connector lines instead of bezier curves:

    ```
    Parent Node
        |           <- vertical line down from parent bottom-center
        |
    ----+----       <- horizontal line spanning all children
    |       |
    |       |       <- vertical lines down to each child top-center
    Child1  Child2
    ```

    For each parent-children group (NOT each individual connection), draw:
    1. A vertical line from the parent's bottom-center down to a midpoint
    2. A horizontal line at the midpoint spanning from the leftmost child to the rightmost child
    3. For each child, a vertical line from the horizontal line down to the child's top-center

    The midpoint Y should be halfway between the parent's bottom and the children's top.

    Change the interface to accept parent-children GROUPS instead of individual connections:
    ```typescript
    interface ConnectionGroup {
      parentId: string;
      childIds: string[];
    }

    interface AgentTreeLinesProps {
      nodePositions: Map<string, DOMRect>;
      connectionGroups: ConnectionGroup[];
      containerRect: DOMRect | null;
    }
    ```

    Implementation:
    ```tsx
    export function AgentTreeLines({
      nodePositions,
      connectionGroups,
      containerRect,
    }: AgentTreeLinesProps) {
      if (!containerRect || connectionGroups.length === 0) return null;

      const elements: JSX.Element[] = [];

      for (const group of connectionGroups) {
        const parentRect = nodePositions.get(group.parentId);
        if (!parentRect) continue;

        // Parent bottom center (relative to container)
        const px = parentRect.left - containerRect.left + parentRect.width / 2;
        const py = parentRect.top - containerRect.top + parentRect.height;

        // Collect child top centers
        const childPoints: { x: number; y: number; id: string }[] = [];
        for (const childId of group.childIds) {
          const childRect = nodePositions.get(childId);
          if (!childRect) continue;
          childPoints.push({
            x: childRect.left - containerRect.left + childRect.width / 2,
            y: childRect.top - containerRect.top,
            id: childId,
          });
        }

        if (childPoints.length === 0) continue;

        // Midpoint Y between parent bottom and first child top
        const midY = py + (childPoints[0].y - py) / 2;

        // 1. Vertical line from parent down to midpoint
        elements.push(
          <line key={`v-${group.parentId}`}
            x1={px} y1={py} x2={px} y2={midY}
            stroke="currentColor" className="text-border" strokeWidth={1.5} />
        );

        if (childPoints.length === 1) {
          // Single child: straight vertical line through midpoint to child
          elements.push(
            <line key={`vc-${childPoints[0].id}`}
              x1={px} y1={midY} x2={childPoints[0].x} y2={childPoints[0].y}
              stroke="currentColor" className="text-border" strokeWidth={1.5} />
          );
        } else {
          // Multiple children: horizontal bar + verticals to each
          const leftX = Math.min(...childPoints.map(c => c.x));
          const rightX = Math.max(...childPoints.map(c => c.x));

          // 2. Horizontal line at midpoint
          elements.push(
            <line key={`h-${group.parentId}`}
              x1={leftX} y1={midY} x2={rightX} y2={midY}
              stroke="currentColor" className="text-border" strokeWidth={1.5} />
          );

          // 3. Vertical lines from horizontal bar down to each child
          for (const child of childPoints) {
            elements.push(
              <line key={`vc-${child.id}`}
                x1={child.x} y1={midY} x2={child.x} y2={child.y}
                stroke="currentColor" className="text-border" strokeWidth={1.5} />
            );
          }
        }
      }

      return (
        <svg className="pointer-events-none absolute inset-0 z-0"
          width={containerRect.width} height={containerRect.height}>
          {elements}
        </svg>
      );
    }
    ```
