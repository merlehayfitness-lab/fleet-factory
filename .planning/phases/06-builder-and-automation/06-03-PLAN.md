<objective>
Replace mock tool execution and stub chat responses with real VPS agent routing, with graceful degradation when VPS is offline.

Purpose: This plan connects the admin app to live agents on the VPS. The chat service switches from generating stub responses to sending messages to real VPS agents and streaming back their responses. The task executor switches from mock tool execution to submitting tasks to real VPS agents and receiving actual results. Both paths check VPS health first and fall back gracefully to the existing stub/mock behavior when the VPS is unreachable or not configured.

Output: VPS chat routing service, VPS task routing service, evolved chat-service.ts with VPS-aware routing, evolved executor.ts with VPS-aware execution, WebSocket chat streaming support, graceful degradation for offline VPS, inter-agent messaging config.
</objective>
