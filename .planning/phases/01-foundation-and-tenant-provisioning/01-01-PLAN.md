    - role (text, NOT NULL, CHECK in ('owner', 'admin', 'manager', 'member'))
    - created_at
    - UNIQUE constraint on (business_id, user_id)
    - Enable RLS
    - Composite index on (business_id, user_id)

    **004_departments.sql:**

    - created_at, updated_at
    - Enable RLS
    - Composite index on (business_id, department_id)

    - config_snapshot (jsonb)
    - error_message (text)
    - started_at, completed_at (timestamptz)
    - created_at
    - Enable RLS
    - Index on (business_id, created_at DESC)

    **008_audit_logs.sql:**
    - id (uuid, default gen_random_uuid(), PK)

    - actor_id (uuid, REFERENCES auth.users)
    - action (text, NOT NULL)
    - entity_type (text)
    - entity_id (uuid)
    - metadata (jsonb, default '{}')
    - created_at (timestamptz, default now())
    - Enable RLS
    - Index on (business_id, created_at DESC)
    - Index on (business_id, action)

    **009_rls_helpers.sql:**
    Create two SECURITY DEFINER functions (following the research pattern exactly):

    1. `is_business_member(p_business_id uuid) RETURNS boolean`: Checks if auth.uid() is in business_users for the given business AND business status != 'disabled'. Use `(SELECT auth.uid())` for query plan caching. SET search_path = ''.

    2. `has_role_on_business(p_business_id uuid, p_role text DEFAULT NULL) RETURNS boolean`: Same as above but also checks role matches. If p_role is NULL, any role matches (acts like is_business_member).

    Use CREATE OR REPLACE FUNCTION for idempotency.

    **010_rls_policies.sql:**
    Create RLS policies for EVERY table:

    - **businesses:** SELECT if is_business_member(id). INSERT for authenticated (any user can create). UPDATE/DELETE only owner via has_role_on_business(id, 'owner').
    - **profiles:** SELECT/UPDATE own profile only (auth.uid() = id).
    - **business_users:** SELECT if is_business_member(business_id). INSERT/UPDATE/DELETE only owner via has_role_on_business.
    - **departments:** SELECT if is_business_member(business_id). INSERT/UPDATE/DELETE owner/admin via has_role_on_business.
    - **agent_templates:** SELECT for all authenticated (USING (true)). No INSERT/UPDATE/DELETE policies (admin via service_role only).
    - **agents:** SELECT if is_business_member(business_id). INSERT/UPDATE/DELETE owner/admin.
    - **deployments:** SELECT if is_business_member(business_id). INSERT/UPDATE owner/admin.
    - **audit_logs:** SELECT if is_business_member(business_id). INSERT for authenticated (actions log themselves). No UPDATE/DELETE.

    CRITICAL: Every table MUST have ENABLE ROW LEVEL SECURITY in its own CREATE TABLE file AND policies in 010_rls_policies.sql. The kill switch works because is_business_member checks `b.status != 'disabled'` -- when a business is disabled, ALL RLS checks fail and no data is accessible (ISOL-05).
