  <action>
    **packages/db/schema/010_provision_rpc.sql:**
    Create the provision_business_tenant() function following the exact pattern from research:

    ```sql
    CREATE OR REPLACE FUNCTION public.provision_business_tenant(
      p_name text,
      p_slug text,
      p_industry text DEFAULT 'general'
    )
    RETURNS uuid
    LANGUAGE plpgsql
    SECURITY DEFINER
    SET search_path = ''
    ```

    The function must:
    1. Get auth.uid() into v_user_id (fail if null -- must be authenticated)
    2. **Idempotency check (PROV-06):** Look for an existing business with the same slug where the current user is owner. If found, RETURN the existing business_id immediately.
    3. **Create business** with status = 'provisioning'
    4. **Create owner membership** in business_users (role = 'owner')
    5. **Seed 4 departments** (Owner, Sales, Support, Operations) using FOREACH over text array
    6. **Create agents from templates:** For each department, query agent_templates WHERE department_type matches AND is_active = true. INSERT matching agents with the template's system_prompt, tool_profile, model_profile. Agent status = 'provisioning'.
    7. **Queue deployment:** INSERT into deployments with version=1, status='queued'
    8. **Update business status** to 'active'
    9. RETURN the business_id

    The entire function body is wrapped in a transaction automatically by Postgres. If any step fails, everything rolls back (PROV-02).

    **packages/core/types/index.ts:**
    Export TypeScript types for the domain:
    - `BusinessStatus = 'provisioning' | 'active' | 'suspended' | 'disabled'`
    - `UserRole = 'owner' | 'admin' | 'manager' | 'member'`
    - `DepartmentType = 'owner' | 'sales' | 'support' | 'operations' | 'custom'`
    - `AgentStatus = 'provisioning' | 'active' | 'paused' | 'error' | 'retired'`
    - `DeploymentStatus = 'queued' | 'building' | 'deploying' | 'live' | 'failed' | 'rolled_back'`

    **packages/core/tenant/schema.ts:**
    Export Zod schemas using Zod v4:
    - `createBusinessSchema`: validates { name: string (2-100 chars), slug: string (2-50 chars, lowercase alphanumeric + hyphens only via regex /^[a-z0-9-]+$/), industry: string (1+ chars, default 'general') }
    - Export the inferred type: `type CreateBusinessInput = z.infer<typeof createBusinessSchema>`

    **packages/core/tenant/provision.ts:**
    Export a `provisionBusinessTenant` function that:
    - Takes a Supabase client and a validated CreateBusinessInput
    - Calls supabase.rpc('provision_business_tenant', { p_name, p_slug, p_industry })
    - Returns the business_id (uuid string) on success
    - Throws a descriptive error on failure
    - This is a thin wrapper -- all logic is in the RPC. Keeps Server Actions under 15 lines per research anti-pattern guidance.

    **packages/core/index.ts:**
    Re-export from tenant/schema and tenant/provision and types/index.

    Install zod in packages/core: `pnpm --filter @fleet-factory/core add zod`
    Add @fleet-factory/core as a workspace dependency in apps/web: `pnpm --filter web add @fleet-factory/core@workspace:*`
