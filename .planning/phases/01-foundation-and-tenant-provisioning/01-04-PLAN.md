    **apps/web/app/(dashboard)/businesses/page.tsx:**
    A Server Component that:
    - Calls createServerClient() and getUser() (redirect if no user)
    - Queries businesses with their user role: `supabase.from('businesses').select('*, business_users!inner(role)').order('created_at', { ascending: false })`
    - RLS automatically scopes to user's businesses (NO manual filtering)
    - If no businesses, show empty state with a "Create your first business" link to /businesses/new
    - Otherwise render BusinessList component
    - Include a "New Business" button/link in the header that goes to /businesses/new

    **apps/web/_components/business-list.tsx:**
    A client or server component that renders a table/card grid of businesses:
    - Use shadcn Table component
    - Columns: Name (link to /businesses/[id]), Industry, Status (StatusBadge), Role, Created date (formatted)
    - Each row links to the business overview page
    - Clean, readable table per CLAUDE.md UI style

    **apps/web/app/(dashboard)/businesses/[id]/page.tsx:**
    A Server Component business overview dashboard:
    - Fetch business details: `supabase.from('businesses').select('*').eq('id', params.id).single()`
    - Fetch agent count: `supabase.from('agents').select('id', { count: 'exact', head: true }).eq('business_id', params.id)`
    - Fetch department count: `supabase.from('departments').select('id', { count: 'exact', head: true }).eq('business_id', params.id)`
    - Fetch latest deployment: `supabase.from('deployments').select('*').eq('business_id', params.id).order('created_at', { ascending: false }).limit(1).single()`
    - Fetch pending approval count: 0 for now (approvals table doesn't exist yet in Phase 1)
    - If business not found, call notFound()
    - Render BusinessOverview component with all this data

    **apps/web/_components/business-overview.tsx:**
    Dashboard card layout showing (per CLAUDE.md requirements):
    - **Business name** as page title with status badge
    - **Stats cards** in a responsive grid (grid-cols-1 sm:grid-cols-2 lg:grid-cols-4):
      - Deployment Status (StatusBadge of latest deployment, or "No deployments" if none)
      - Active Agents (count number)
      - Departments (count number)
      - Pending Approvals (count number, "0" for now)
    - **Quick Links** section with cards/buttons:
      - Departments (/businesses/[id]/departments)
      - Deployments (grayed out -- Phase 3)
      - Tasks (grayed out -- Phase 4)
      - Logs (grayed out -- Phase 5)
    - **Recent Activity** section: placeholder "No recent activity" for now (audit_logs will be populated in later phases)
    - Use shadcn Card components for each section
    - Use Lucide icons for visual indicators

    **apps/web/app/(dashboard)/businesses/[id]/departments/page.tsx:**
    A Server Component that:
    - Fetches departments for the business: `supabase.from('departments').select('*').eq('business_id', params.id).order('created_at')`
    - Renders DepartmentsList component
    - Shows empty state if no departments (shouldn't happen after provisioning, but handle gracefully)

    **apps/web/_components/departments-list.tsx:**
    Renders a grid of department cards (or a table):
    - Each department shows: Name, Type (with icon per type: Crown for owner, TrendingUp for sales, Headphones for support, Settings for operations), Description, Created date
    - Use shadcn Card components in a responsive grid (grid-cols-1 md:grid-cols-2)
    - Each card shows the department name prominently, type as a subtle label, and description below
    - No edit/delete functionality for MVP (departments are seeded and managed by the system)

    IMPORTANT: All queries use the Supabase client with the anon key (via createServerClient). RLS automatically scopes results to the user's businesses. Do NOT add .eq('business_id', ...) as a security measure -- RLS handles that. The .eq() is for fetching specific business data within the user's allowed scope, not for security.
