    **packages/db/schema/036_skills_tables.sql:**
    Create migration with three tables:

    1. `skills` table:
    ```sql
    CREATE TABLE IF NOT EXISTS public.skills (
      id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
      business_id uuid NOT NULL REFERENCES public.businesses ON DELETE CASCADE,
      name text NOT NULL,
      description text,
      content text NOT NULL,
      trigger_phrases text[],
      source_type text NOT NULL DEFAULT 'manual'
        CHECK (source_type IN ('manual', 'imported', 'template')),
      source_url text,
      version integer NOT NULL DEFAULT 1,
      deleted_at timestamptz,
      created_at timestamptz DEFAULT now() NOT NULL,
      updated_at timestamptz DEFAULT now() NOT NULL
    );
    ```
    With indexes: `idx_skills_business` on (business_id), `idx_skills_business_name` on (business_id, name).

    2. `skill_assignments` table:
    ```sql
    CREATE TABLE IF NOT EXISTS public.skill_assignments (
      id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
      skill_id uuid NOT NULL REFERENCES public.skills ON DELETE CASCADE,
      business_id uuid NOT NULL REFERENCES public.businesses ON DELETE CASCADE,
      agent_id uuid REFERENCES public.agents ON DELETE CASCADE,
      department_id uuid REFERENCES public.departments ON DELETE CASCADE,
      created_at timestamptz DEFAULT now() NOT NULL,
      CONSTRAINT skill_assignment_target CHECK (
        (agent_id IS NOT NULL AND department_id IS NULL)
        OR (agent_id IS NULL AND department_id IS NOT NULL)
      )
    );
    ```
    With indexes: `idx_skill_assignments_agent`, `idx_skill_assignments_department`, `idx_skill_assignments_skill`. Unique partial indexes: `idx_skill_assignments_unique_agent` on (skill_id, agent_id) WHERE agent_id IS NOT NULL, `idx_skill_assignments_unique_dept` on (skill_id, department_id) WHERE department_id IS NOT NULL.

    3. `skill_templates` table:
    ```sql
    CREATE TABLE IF NOT EXISTS public.skill_templates (
      id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
      name text NOT NULL,
      description text,
      content text NOT NULL,
      department_type text NOT NULL,
      role_type text,
      trigger_phrases text[],
      is_active boolean DEFAULT true,
      created_at timestamptz DEFAULT now() NOT NULL,
      updated_at timestamptz DEFAULT now() NOT NULL
    );
    ```
    With index: `idx_skill_templates_department` on (department_type).

    RLS policies:
    - skills: SELECT for members (is_business_member), INSERT/UPDATE/DELETE for owner/admin (has_role_on_business)
    - skill_assignments: SELECT for members, INSERT/DELETE for owner/admin
    - skill_templates: SELECT for all authenticated users (globally readable)

    Seed 10 starter skill templates (2-3 per department):
    - **Owner**: "Strategic Planning" (goal setting, KPI tracking), "Team Coordination" (cross-dept communication)
    - **Sales**: "Lead Qualification" (ICP matching, scoring), "Outreach Cadence" (email sequences, follow-ups), "Deal Management" (pipeline tracking)
    - **Support**: "Ticket Triage" (classification, routing), "Knowledge Base Lookup" (search, answer synthesis)
    - **Operations**: "Process Automation" (workflow triggers, reporting), "Data Analysis" (metrics, anomaly detection), "Resource Scheduling" (calendar, capacity)

    Each template INSERT should include name, description, content (full SKILL.md body with ## Capabilities, ## Workflows, ## Boundaries sections), department_type, role_type, and trigger_phrases array.

    **packages/core/skill/skill-service.ts:**
    Skill CRUD and assignment operations. All functions take SupabaseClient as first argument (matching existing service patterns):

    - `createSkill(supabase, businessId, data: {name, description?, content, trigger_phrases?, source_type?, source_url?})` -- INSERT into skills, return skill
    - `updateSkill(supabase, skillId, businessId, data: {name?, description?, content?, trigger_phrases?})` -- UPDATE skill, increment version, update updated_at, return skill
    - `softDeleteSkill(supabase, skillId, businessId)` -- SET deleted_at = now(), return void
    - `getSkill(supabase, skillId)` -- SELECT single skill, return Skill | null
    - `listSkillsForBusiness(supabase, businessId)` -- SELECT skills WHERE business_id AND deleted_at IS NULL, ordered by name
    - `assignSkill(supabase, skillId, businessId, target: {agent_id?: string, department_id?: string})` -- INSERT into skill_assignments, return assignment
    - `unassignSkill(supabase, assignmentId)` -- DELETE from skill_assignments
    - `getSkillsForAgent(supabase, agentId, departmentId)` -- Returns SkillWithAssignment[]. Query: direct agent assignments UNION department assignments for agent's department. Include assignment_level field ('agent' or 'department').
    - `getSkillsForDepartment(supabase, departmentId)` -- SELECT skills via skill_assignments WHERE department_id, return Skill[]
    - `getSkillUsage(supabase, skillId)` -- COUNT DISTINCT agent_id and department_id from skill_assignments, plus joined names. Return SkillUsage.
    - `getSkillTemplates(supabase, departmentType?: string)` -- SELECT from skill_templates WHERE is_active, optionally filtered by department_type. Return SkillTemplate[].
    - `createSkillFromTemplate(supabase, businessId, templateId)` -- Fetch template, INSERT new skill with source_type='template', return skill.

    **packages/core/skill/skill-compiler.ts:**
    Compiles multiple assigned skills into a single SKILL.md:

    - `compileSkills(agentName: string, departmentSkills: Skill[], agentSkills: Skill[]): CompiledSkill`

    Logic:
    1. Department skills go under `## Department Skills` heading with each skill as `### {skill.name}` subsection
    2. Agent skills go under `## Agent Skills` heading with each skill as `### {skill.name}` subsection
    3. Name conflict resolution: if an agent skill has the same name as a department skill, the department skill is skipped (agent takes precedence) with a comment noting the override
    4. Frontmatter with name and description
    5. Enforce 4000 char budget with truncation marker
    6. Return CompiledSkill with content and sources array

    **packages/core/skill/github-import.ts:**
    GitHub URL parsing and content fetching:

    - `parseGitHubUrl(url: string): GitHubUrlInfo | null` -- Parse GitHub URLs in formats: `github.com/{owner}/{repo}/blob/{branch}/{path}` (file), `github.com/{owner}/{repo}/tree/{branch}/{path}` (directory). Extract owner, repo, path, branch, type. Return null for invalid URLs.
    - `fetchGitHubFile(info: GitHubUrlInfo): Promise<GitHubImportResult>` -- Fetch raw content from `raw.githubusercontent.com/{owner}/{repo}/{branch}/{path}`. Extract name from filename (without .md extension). Return {name, content, source_url}.
    - `fetchGitHubDirectory(info: GitHubUrlInfo): Promise<GitHubImportResult[]>` -- Use GitHub API `api.github.com/repos/{owner}/{repo}/contents/{path}?ref={branch}` to list files, filter .md files, fetch each via fetchGitHubFile. Return array of results.

    Include error handling: timeout (10s via AbortController), non-200 status codes, rate limiting detection (403 with rate limit headers).
