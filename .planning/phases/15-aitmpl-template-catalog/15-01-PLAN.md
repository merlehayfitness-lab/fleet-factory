    **packages/core/aitmpl/catalog-types.ts** (NEW):
    Define TypeScript interfaces matching the AITMPL JSON schema from research.

    ```typescript
    /** The 7 AITMPL component types. */
    export type AitmplComponentType = "skill" | "agent" | "command" | "mcp" | "setting" | "hook" | "plugin";
    ```

    `AitmplComponent` interface:
    - `name: string` — component name (e.g., "ai-ethics-advisor")
    - `path: string` — catalog path (e.g., "ai-specialists/ai-ethics-advisor.md")
    - `category: string` — AITMPL category (e.g., "ai-specialists")
    - `type: string` — component type ("agent", "skill", etc.)
    - `content: string` — full markdown or JSON content
    - `description: string` — brief description
    - `author: string` — usually empty
    - `repo: string` — usually empty
    - `version: string` — usually empty
    - `license: string` — usually empty
    - `keywords: string[]` — usually empty
    - `downloads: number` — download count from CLI usage
    - `security: { validated: boolean; valid: boolean | null; score: number | null; errorCount: number; warningCount: number; lastValidated: string | null }`

    `AitmplPlugin` interface (different shape from components):
    - `name: string`
    - `id: string`
    - `type: "plugin"`
    - `description: string`
    - `version: string`
    - `keywords: string[]`
    - `author: string`
    - `commands: number` — count of commands
    - `agents: number` — count of agents
    - `mcpServers: number` — count of MCP servers
    - `commandsList: string[]` — paths like "data-ai/ai-engineer"
    - `agentsList: string[]`
    - `mcpServersList: string[]`
    - `installCommand: string`
    - `downloads: number`

    `AitmplTemplate` interface (project scaffolds, included for completeness):
    - `name: string`
    - `id: string`
    - `type: "template"`
    - `subtype: string`
    - `category: string`
    - `language: string`
    - `description: string`
    - `files: string[]`
    - `installCommand: string`
    - `downloads: number`

    `AitmplCatalog` interface — the full JSON structure:
    - `skills: AitmplComponent[]`
    - `agents: AitmplComponent[]`
    - `commands: AitmplComponent[]`
    - `mcps: AitmplComponent[]`
    - `settings: AitmplComponent[]`
    - `hooks: AitmplComponent[]`
    - `plugins: AitmplPlugin[]`
    - `templates: AitmplTemplate[]`

    `CatalogSearchResult` — lightweight (no content field):
    - `name: string`
    - `path: string`
    - `category: string`
    - `type: string`
    - `description: string`
    - `downloads: number`

    `AitmplImportOptions` interface:
    - `businessId: string`
    - `componentPath: string`
    - `componentType: AitmplComponentType`
    - `targetAgentId?: string`
    - `targetDepartmentId?: string`

    `AitmplImportResult` interface:
    - `success: boolean`
    - `entityType: "skill" | "agent_prompt" | "tool_profile_mcp"`
    - `entityId?: string` — ID of the created/modified entity
    - `name: string`
    - `error?: string`

    **packages/core/aitmpl/catalog-service.ts** (NEW):
    Server-side catalog fetch with module-level TTL caching.

    Module-level cache variables:
    ```typescript
    let cachedData: AitmplCatalog | null = null;
    let cacheTimestamp = 0;
    const CACHE_TTL = 24 * 60 * 60 * 1000; // 24 hours
    const CATALOG_URL = "https://www.aitmpl.com/components.json";
    ```

    Export `async function getCatalog(): Promise<AitmplCatalog>`:
    - If `cachedData` exists and `Date.now() - cacheTimestamp < CACHE_TTL`, return cached
    - Otherwise fetch from `CATALOG_URL` with `{ next: { revalidate: 86400 } }` option
    - Parse as JSON, store in `cachedData`, update `cacheTimestamp`
    - Wrap in try/catch: on failure, if `cachedData` exists return stale cache, otherwise throw
    - Return the catalog

    Export `function clearCatalogCache(): void`:
    - Set `cachedData = null` and `cacheTimestamp = 0`

    Type alias for search options:
    ```typescript
    interface SearchOptions {
      type?: AitmplComponentType;
      department?: string;
      limit?: number;
      sort?: "downloads" | "name" | "newest";
    }
    ```

    Export `async function searchComponents(query: string, options?: SearchOptions): Promise<CatalogSearchResult[]>`:
    - Call `getCatalog()`
    - Determine which array(s) to search: if `options?.type` is set, pick `catalog[type + 's']` (handle "mcp" -> "mcps"). For "plugin" type, use `catalog.plugins` but map to CatalogSearchResult shape.
    - If no type, search ALL component arrays (skills, agents, commands, mcps, settings, hooks) — skip plugins and templates for unified search
    - Filter with `String.toLowerCase().includes()` on `name`, `description`, and `category`
    - If `options?.department`, get `DEPARTMENT_CATEGORY_MAP[department]` and additionally filter to items whose category is in that list. But don't exclude non-matching items entirely — instead, put matching items first (recommended) and then non-matching items after.
    - Sort: "downloads" (default) = `b.downloads - a.downloads`, "name" = `a.name.localeCompare(b.name)`, "newest" = just use download order as proxy (no date field available)
    - Slice to `options?.limit ?? 50`
    - Map to `CatalogSearchResult` (strip content field): `{ name, path, category, type, description, downloads }`

    Export `async function getComponentDetail(path: string, type: AitmplComponentType): Promise<AitmplComponent | null>`:
    - Call `getCatalog()`
    - Look up in `catalog[type + 's']` and find by `path`
    - Return full component with content, or null if not found

    Export `async function getComponentsByType(type: AitmplComponentType, limit?: number): Promise<CatalogSearchResult[]>`:
    - Shorthand: calls `searchComponents("", { type, limit })`

    Export `async function getCatalogStats(): Promise<Record<AitmplComponentType, number>>`:
    - Returns count of items per type from catalog arrays

    **packages/core/aitmpl/import-service.ts** (NEW):
    Import logic converting AITMPL components into Agency Factory entities.

    Import dependencies:
    ```typescript
    import type { SupabaseClient } from "@supabase/supabase-js";
    import { getComponentDetail, getCatalog } from "./catalog-service";
    import type { AitmplImportOptions, AitmplImportResult, AitmplComponent, AitmplPlugin } from "./catalog-types";
    import { createSkill, assignSkill } from "../skill/skill-service";
    ```

    Export `async function importFromAitmpl(supabase: SupabaseClient, options: AitmplImportOptions): Promise<AitmplImportResult>`:

    1. Call `getComponentDetail(options.componentPath, options.componentType)` to get full content
    2. If not found, return `{ success: false, entityType: "skill", name: options.componentPath, error: "Component not found in AITMPL catalog" }`
    3. Switch on `options.componentType`:

    **Case "skill" | "command" | "setting" | "hook"** (all imported as skills per 15-CONTEXT.md):
    - Call `createSkill(supabase, options.businessId, { name: component.name, description: component.description, content: component.content, source_type: "imported", source_url: \`aitmpl://${options.componentType}/${component.path}\` })`
    - If `options.targetAgentId`, call `assignSkill(supabase, skill.id, options.businessId, { agent_id: options.targetAgentId })`
    - If `options.targetDepartmentId` (and no agent), call `assignSkill(supabase, skill.id, options.businessId, { department_id: options.targetDepartmentId })`
    - Return `{ success: true, entityType: "skill", entityId: skill.id, name: component.name }`

    **Case "agent"** (content imported as system_prompt):
    - If `options.targetAgentId` is provided:
      - Update agent's system_prompt: `supabase.from("agents").update({ system_prompt: component.content, updated_at: new Date().toISOString() }).eq("id", options.targetAgentId)`
      - Return `{ success: true, entityType: "agent_prompt", entityId: options.targetAgentId, name: component.name }`
    - If no target agent, create a skill instead (fallback):
      - Same as skill case but note it in description: `"AITMPL agent template: ${component.description}"`
      - Return `{ success: true, entityType: "skill", entityId: skill.id, name: component.name }`

    **Case "mcp"** (merge into tool_profile.mcp_servers[]):
    - If no `options.targetAgentId`, return error: `"MCP import requires a target agent"`
    - Parse `component.content` as JSON with try/catch: `const mcpConfig = JSON.parse(component.content)`
    - Fetch current agent: `supabase.from("agents").select("tool_profile").eq("id", options.targetAgentId).single()`
    - Existing profile: `const profile = (agent.tool_profile ?? { allowed_tools: ["*"], mcp_servers: [] }) as ToolProfileShape`
    - Build new MCP server entry from parsed config. The AITMPL MCP content varies, so extract: `{ name: mcpConfig.name || component.name, url: mcpConfig.command || mcpConfig.url || "", transport: mcpConfig.transport || "stdio", env: mcpConfig.env || {}, enabled: true }`
    - Append to `profile.mcp_servers`
    - Update: `supabase.from("agents").update({ tool_profile: profile, updated_at: new Date().toISOString() }).eq("id", options.targetAgentId)`
    - Return `{ success: true, entityType: "tool_profile_mcp", entityId: options.targetAgentId, name: component.name }`

    **Case "plugin"** (decompose — per CONTEXT.md plugins are imported via their constituent parts):
    - The catalog stores plugins in `catalog.plugins` as `AitmplPlugin` (not `AitmplComponent`). Since `getComponentDetail` searches component arrays by path, it won't find a plugin. Handle at the top of the function: if `options.componentType === "plugin"`, skip the `getComponentDetail` call. Instead, call `getCatalog()` and find the plugin by matching `options.componentPath` against `plugin.id` in `catalog.plugins`.
    - Extract `commandsList`, `agentsList`, `mcpServersList` from the plugin.
    - Return `{ success: false, entityType: "skill", name: plugin?.name ?? options.componentPath, error: \`Plugin import requires decomposition. Import its parts individually:\n${agentsList.map(a => \`  - Agent: ${a}\`).join('\\n')}\n${commandsList.map(c => \`  - Command: ${c}\`).join('\\n')}\n${mcpServersList.map(m => \`  - MCP: ${m}\`).join('\\n')}\` }`
    - This gives the UI actionable info about which constituent parts to import separately.

    **Default**: Return error for unknown type.

    Wrap entire function body in try/catch. On error: return `{ success: false, entityType: "skill", name: options.componentPath, error: err.message }`
