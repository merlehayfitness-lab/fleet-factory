  <files>apps/web/_components/aitmpl-suggestion-banner.tsx</files>
  <action>
In `aitmpl-suggestion-banner.tsx`, the useEffect that reads localStorage on mount and the `dismiss()` function both use `storageKey(businessId)` without checking if businessId is a valid non-empty string. When the component mounts before businessId hydrates (or if businessId is momentarily undefined), the key becomes `aitmpl-banner-dismissed-undefined`, and subsequent renders with the real businessId find no stored value.

Fix:

1. In the useEffect (line ~28-31), add a guard at the top: `if (!businessId) return;` — this prevents reading/writing with an invalid key. The effect will re-run when businessId becomes available because it's in the dependency array.

2. In the `dismiss()` function (line ~33-36), add a guard at the top: `if (!businessId) return;` — this prevents writing the dismiss state under an invalid key.

No other changes needed. The storageKey helper, default `dismissed=true` SSR flash prevention, and auto-dismiss on import all remain as-is.
  </action>

  </verify>

The AITMPL catalog target picker shows raw UUIDs because the skills page doesn't fetch agents/departments and SkillLibrary doesn't forward them to SkillTemplateBrowser.

**In `apps/web/app/(dashboard)/businesses/[id]/skills/page.tsx`:**

1. After the existing business and skills queries, add two more Supabase queries (use Promise.all for parallel execution):
   - Fetch agents: `supabase.from("agents").select("id, name, departments(name)").eq("business_id", id).is("deleted_at", null).order("name")` — then map each agent to `{ id, name, department_name: agent.departments?.name }` (departments is a joined object, not array, because it's a belongsTo).
   - Fetch departments: `supabase.from("departments").select("id, name").eq("business_id", id).order("name")`

2. Pass the fetched `agents` and `departments` arrays as props to `<SkillLibrary>`:
   ```
   <SkillLibrary
     businessId={id}
     initialSkills={(skills ?? []) as unknown as Skill[]}
     agents={agents}
     departments={departments}
   />
   ```

**In `apps/web/_components/skill-library.tsx`:**

1. Update the `SkillLibraryProps` interface to accept optional `agents` and `departments`:
   ```
   agents?: Array<{ id: string; name: string; department_name?: string }>;
   departments?: Array<{ id: string; name: string }>;
   ```

2. Destructure these new props in the component function.

3. Pass them through to `<SkillTemplateBrowser>` which already accepts these props:
   ```
   <SkillTemplateBrowser
     businessId={businessId}
     onSkillAdded={() => { refetchSkills(); }}
     open={templateBrowserOpen}
     onOpenChange={setTemplateBrowserOpen}
     agents={agents}
     departments={departments}
   />
   ```

The SkillTemplateBrowser already has `agents` and `departments` in its props interface (lines 36-37 of skill-template-browser.tsx) and forwards them to AitmplCatalogBrowser (line 289-290). The chain just needs data flowing from the server page down.
