Database schema for Slack integration tables, core Slack service modules (client, events, channels, messages, OAuth), API route handlers for Events API and OAuth flow, and server actions for frontend interaction.

  <action>
    Create migration 041_slack_tables.sql with the following:

    **1. slack_installations table:**
    - id (uuid PK DEFAULT gen_random_uuid())
    - business_id (uuid NOT NULL REFERENCES businesses ON DELETE CASCADE)
    - slack_team_id (text NOT NULL) -- Slack workspace ID
    - slack_team_name (text) -- Workspace display name
    - bot_user_id (text NOT NULL) -- Bot's Slack user ID (for filtering own messages)
    - installed_at (timestamptz DEFAULT now() NOT NULL)
    - UNIQUE(business_id) -- one Slack workspace per business
    - UNIQUE(slack_team_id) -- one business per Slack workspace
    - Enable RLS with SELECT/INSERT/UPDATE policies gated by is_business_member(business_id)
    - Index on slack_team_id for event lookup

    **2. slack_channel_mappings table:**
    - id (uuid PK DEFAULT gen_random_uuid())
    - business_id (uuid NOT NULL REFERENCES businesses ON DELETE CASCADE)
    - department_id (uuid NOT NULL REFERENCES departments ON DELETE CASCADE)
    - agent_id (uuid REFERENCES agents ON DELETE SET NULL) -- optional: for sub-agent channels
    - slack_channel_id (text NOT NULL)
    - slack_channel_name (text NOT NULL)
    - created_at (timestamptz DEFAULT now() NOT NULL)
    - UNIQUE index on (business_id, slack_channel_id)
    - UNIQUE index on (business_id, department_id) WHERE agent_id IS NULL (one main channel per dept)
    - Enable RLS with SELECT/INSERT/UPDATE/DELETE policies gated by is_business_member(business_id)

    **3. ALTER messages table:**
    - ADD COLUMN IF NOT EXISTS slack_ts text -- Slack message timestamp (for dedup and thread linking)
    - ADD COLUMN IF NOT EXISTS slack_channel_id text -- Slack channel where message originated
    - CREATE INDEX IF NOT EXISTS idx_messages_slack_ts ON messages (slack_ts) WHERE slack_ts IS NOT NULL

    **4. ALTER conversations table:**
    - ADD COLUMN IF NOT EXISTS slack_channel_id text -- Link conversation to Slack channel

    Append the migration to _combined_schema.sql.
  </action>

    - SlackChannelMapping interface: { id, businessId, departmentId, agentId, slackChannelId, slackChannelName, createdAt }
    - SlackEventPayload interface: { type, token, team_id, event, event_id, event_time, ... }
    - SlackMessageEvent interface: { type, channel, user, text, ts, thread_ts, bot_id, subtype, ... }
    - SlackConnectionStatus type: { connected: boolean, teamName, teamId, botUserId, installedAt } | { connected: false }
    - Export all from packages/core/index.ts (client-safe types only)

    **Create packages/core/slack/slack-client.ts:**
    - `createSlackClient(botToken: string): WebClient` -- simple factory, no DB dependency
    - `getSlackClient(supabase, businessId): Promise<WebClient | null>` -- fetches bot_token from secrets table (provider='slack', key='bot_token'), decrypts it using decrypt() from crypto/encryption.ts, returns WebClient or null if no token found
    - Pattern: uses the existing secrets/encryption infrastructure from Phase 13

    **Create packages/core/slack/slack-events.ts:**
    - `verifySlackSignature(signingSecret, rawBody, timestamp, signature): boolean` -- HMAC-SHA256 verification with replay protection (reject if timestamp > 5 min old), uses timing-safe comparison
    - `parseSlackEvent(rawBody: string): SlackEventPayload` -- JSON parse with basic type assertion
    - `isMessageEvent(event): boolean` -- type guard for message events
    - `isBotMessage(event): boolean` -- returns true if event has bot_id or subtype === 'bot_message' (echo loop prevention)
    - `getSigningSecret(supabase, businessId): Promise<string | null>` -- fetches signing_secret from secrets (provider='slack', key='signing_secret'), decrypts

    **Create packages/core/slack/slack-channels.ts:**
    - `createDepartmentChannels(client: WebClient, supabase, businessId, businessSlug): Promise<SlackChannelMapping[]>` -- fetches all departments for business, creates a Slack channel per department using conversations.create (name pattern: `{slug}-{dept-type}`), handles name_taken by appending suffix, saves mappings to slack_channel_mappings table, returns created mappings
    - `getChannelMappings(supabase, businessId): Promise<SlackChannelMapping[]>` -- fetches all channel mappings for a business
    - `getChannelMapping(supabase, businessId, slackChannelId): Promise<SlackChannelMapping | null>` -- single lookup by Slack channel ID
    - `getDepartmentForChannel(supabase, slackTeamId, slackChannelId): Promise<{ businessId, departmentId } | null>` -- reverse lookup from team_id + channel_id to business + department (for inbound events)
    - `saveChannelMapping(supabase, mapping: Omit<SlackChannelMapping, 'id' | 'createdAt'>): Promise<SlackChannelMapping>` -- insert a mapping
    - Channel name sanitization: lowercase, replace non-alphanumeric/hyphen/underscore with hyphen, max 80 chars

    **Create packages/core/slack/slack-messages.ts:**
    - `handleInboundSlackMessage(supabase, slackTeamId, event: SlackMessageEvent): Promise<void>` -- the main event handler:
      1. Look up business_id and department_id from slack_channel_mappings via getDepartmentForChannel
      2. Skip if no mapping found (message in unmapped channel)
      3. Check for duplicate by querying messages WHERE slack_ts = event.ts (idempotent)
      4. Get or create conversation for this department (reuse getOrCreateConversation from chat-service)
      5. Store inbound message in messages table with role='user', slack_ts=event.ts, slack_channel_id=event.channel
      6. Route to agent via routeAndRespond (existing pipeline)
      7. Post agent response back to Slack via postAgentResponseToSlack
      8. Update the agent message record with the Slack ts returned from chat.postMessage
    - `postAgentResponseToSlack(client: WebClient, channelId, agentName, content, threadTs?): Promise<string | undefined>` -- posts via chat.postMessage with username override, returns message ts
    - `syncMessageToSupabase(supabase, businessId, conversationId, content, role, agentId?, slackTs?, slackChannelId?, metadata?): Promise<void>` -- stores a message with Slack metadata columns
    - Must handle the 3-second event response timeout: all processing is async (void fire-and-forget), event handler returns 200 immediately

    **Create packages/core/slack/slack-oauth.ts:**
    - `getSlackInstallUrl(businessId): string` -- generates the Slack OAuth authorize URL with scopes (channels:manage, channels:read, channels:history, chat:write, users:read, app_mentions:read) and state=businessId, using SLACK_CLIENT_ID env var
    - `handleSlackOAuthCallback(supabase, code, state/businessId): Promise<SlackInstallation>` -- exchanges code for token via oauth.v2.access using SLACK_CLIENT_ID + SLACK_CLIENT_SECRET env vars, stores bot_token and signing_secret via saveProviderCredentials, creates slack_installations record, returns installation
    - `getSlackInstallation(supabase, businessId): Promise<SlackInstallation | null>` -- checks if business has a Slack installation
    - `disconnectSlack(supabase, businessId): Promise<void>` -- deletes slack_installations record, deletes slack_channel_mappings, calls deleteProviderSecrets for slack provider, audit log
    - Env var helpers: `getSlackClientId()`, `getSlackClientSecret()` -- throw on missing (matching project pattern from 01-02)

    **Update packages/core/server.ts:**
    - Add Slack re-exports: getSlackClient, verifySlackSignature, getSigningSecret, createDepartmentChannels, getChannelMappings, handleInboundSlackMessage, postAgentResponseToSlack, getSlackInstallUrl, handleSlackOAuthCallback, getSlackInstallation, disconnectSlack

    **Update packages/core/index.ts:**
    - Re-export client-safe types from slack/slack-types.ts: SlackInstallation, SlackChannelMapping, SlackConnectionStatus
