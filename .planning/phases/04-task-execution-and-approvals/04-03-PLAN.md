    **packages/db/schema/021_approvals_table.sql:**
    Create the approvals table as specified in 04-RESEARCH.md:
    ```sql
    CREATE TABLE IF NOT EXISTS public.approvals (
      id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
      business_id uuid NOT NULL REFERENCES public.businesses ON DELETE CASCADE,
      task_id uuid NOT NULL REFERENCES public.tasks ON DELETE CASCADE,
      agent_id uuid NOT NULL REFERENCES public.agents,
      action_type text NOT NULL,
      action_summary text NOT NULL,
      agent_reasoning text,
      risk_level text NOT NULL CHECK (risk_level IN ('low', 'medium', 'high')),
      risk_explanation text,
      status text NOT NULL DEFAULT 'pending'
        CHECK (status IN ('pending', 'auto_approved', 'approved', 'rejected', 'retry_pending', 'guidance_required')),
      decided_by uuid REFERENCES auth.users,
      decided_at timestamptz,
      decision_note text,
      retry_count integer DEFAULT 0,
      created_at timestamptz DEFAULT now() NOT NULL,
      updated_at timestamptz DEFAULT now() NOT NULL
    );
    ALTER TABLE public.approvals ENABLE ROW LEVEL SECURITY;
    CREATE INDEX IF NOT EXISTS idx_approvals_business ON public.approvals (business_id);
    CREATE INDEX IF NOT EXISTS idx_approvals_task ON public.approvals (task_id);
    CREATE INDEX IF NOT EXISTS idx_approvals_status ON public.approvals (business_id, status);

    CREATE POLICY "approvals_select_member" ON public.approvals FOR SELECT
      TO authenticated USING (public.is_business_member(business_id));
    CREATE POLICY "approvals_insert_admin" ON public.approvals FOR INSERT
      TO authenticated WITH CHECK (
        public.has_role_on_business(business_id, 'owner')
        OR public.has_role_on_business(business_id, 'admin')
        OR public.has_role_on_business(business_id, 'manager'));
    CREATE POLICY "approvals_update_admin" ON public.approvals FOR UPDATE
      TO authenticated
      USING (public.has_role_on_business(business_id, 'owner')
        OR public.has_role_on_business(business_id, 'admin')
        OR public.has_role_on_business(business_id, 'manager'))
      WITH CHECK (public.has_role_on_business(business_id, 'owner')
        OR public.has_role_on_business(business_id, 'admin')
        OR public.has_role_on_business(business_id, 'manager'));
    ```

    CREATE TABLE IF NOT EXISTS public.approval_policies (

      action_pattern text NOT NULL,
      risk_level text NOT NULL CHECK (risk_level IN ('low', 'medium', 'high')),
      description text,

    **packages/core/approval/approval-lifecycle.ts:**
    Approval state machine following deployment/lifecycle.ts pattern:
    ```
    APPROVAL_TRANSITIONS:
      pending -> [auto_approved, approved, rejected]
      auto_approved -> []  (terminal)
      approved -> []  (terminal)
      rejected -> [retry_pending, guidance_required]
      retry_pending -> [pending]
      guidance_required -> [pending]
    ```
    Export: APPROVAL_TRANSITIONS, canTransitionApproval, assertApprovalTransition, getValidApprovalTransitions
    Also add ApprovalStatus type to packages/core/types/index.ts:
    `export type ApprovalStatus = "pending" | "auto_approved" | "approved" | "rejected" | "retry_pending" | "guidance_required";`
    Also add RiskLevel type: `export type RiskLevel = "low" | "medium" | "high";`

    **packages/core/approval/policy-engine.ts:**
    Risk evaluation:
    - `evaluateRisk(supabase, actionType: string): Promise<{ riskLevel: RiskLevel; policy: object | null; explanation: string }>`
      Match actionType against approval_policies.action_pattern using prefix matching. Return matched policy's risk_level and description. If no match, default to 'medium'.
    - `checkAgentTrust(supabase, agentId: string): Promise<boolean>`
      Fetch agent's is_trusted column. Return true if trusted.
    - `shouldAutoApprove(riskLevel: RiskLevel, isTrusted: boolean): boolean`
      Low risk: always auto-approve. Medium risk + trusted agent: auto-approve. Otherwise: needs human review.

    **packages/core/approval/approval-service.ts:**
    Approval CRUD (follow existing service patterns):
    - `createApproval(supabase, businessId, taskId, agentId, actionType, actionSummary, riskLevel, riskExplanation?, agentReasoning?)` -- insert approval with status 'pending' (or 'auto_approved' if shouldAutoApprove), audit log 'approval.created'
    - `getApprovalsForBusiness(supabase, businessId, filters?: { status?, riskLevel? })` -- fetch with joins to tasks(title) and agents(name), ordered by created_at desc
    - `getApprovalById(supabase, approvalId)` -- single fetch with task and agent names
    - `approveAction(supabase, approvalId, decidedBy, decisionNote?)` -- assert transition pending->approved, update decided_by/decided_at/decision_note, transition task back to in_progress, audit log 'approval.approved'
    - `rejectAction(supabase, approvalId, decidedBy, decisionNote?)` -- two-step rejection logic: if retry_count < 1, set status='rejected' then 'retry_pending', increment retry_count. If retry_count >= 1, set status='guidance_required'. Audit log 'approval.rejected'.
    - `provideGuidance(supabase, approvalId, decidedBy, guidance)` -- set decision_note to guidance, transition guidance_required->pending, reset for agent retry. Audit log 'approval.guidance_provided'.
    - `bulkApprove(supabase, approvalIds, decidedBy, decisionNote?)` -- loop approveAction for each ID
    - `bulkReject(supabase, approvalIds, decidedBy, decisionNote?)` -- loop rejectAction for each ID
    All audit logging best-effort.
