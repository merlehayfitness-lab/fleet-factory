    Extend packages/core/secrets/service.ts:
    - Add `getSecretsByProvider(supabase, businessId)`: fetches secrets grouped by provider. Returns Record<string, Secret[]> where key is provider name. Secrets with null provider go into a "legacy" group. Order by provider, then key.
    - Add `revealSecret(supabase, businessId, secretId)`: fetches a single secret by id + business_id, decrypts using decrypt() from crypto/encryption.ts, returns { value: string }. Throws if not found.
    - Add `saveProviderCredentials(supabase, businessId, provider, credentials: Record<string, string>)`: Takes a provider name and a record of field_name -> plaintext value. For each credential entry: encrypt the value, upsert into secrets with business_id, provider, key=field_name, encrypted_value, category='api_key'. Uses the unique index (business_id, provider, key) for upsert via onConflict. After saving all credentials, check if an integration record exists for this provider + business_id. If not, insert one with type derived from INTEGRATION_CATALOG category lookup, status='active', provider set. If integration exists but status is 'inactive' or 'mock', update to 'active'. Audit log: "secret.provider_credentials_saved" with { provider }.
    - Add `deleteProviderSecrets(supabase, businessId, provider)`: Deletes all secrets for a given provider in a business. After deleting, find the integration record for this provider + business_id and set status to 'mock'. Audit log: "secret.provider_credentials_deleted" with { provider }.
    - Keep all existing functions unchanged for backward compatibility.

    Create packages/core/secrets/test-connection.ts:
    - Export `testConnection(provider: string, credentials: Record<string, string>): Promise<{ success: boolean; message: string }>`.
    - For MVP, all providers return mock success: { success: true, message: "Connection verified (mock)" }.
    - Add a comment TODO for each provider where a real API ping could be added later.
    - Structure as a switch/case so real implementations can be slotted in per provider.

    - Re-export testConnection from secrets/test-connection.

    - Add `getAllProviderFieldsAction()`: queries all provider_credential_fields, returns Record<string, CredentialField[]> grouped by provider.
    - Add `revealSecretAction(businessId, secretId)`: auth check, calls revealSecret, returns { value } or { error }.
