Two fixes in this task:

**A) Add UNIQUE constraint to integrations migration (014) and combined schema:**

In `packages/db/schema/014_integrations_table.sql`, add after the existing indexes (after line 22):
```sql
CREATE UNIQUE INDEX IF NOT EXISTS idx_integrations_business_agent_type
  ON public.integrations (business_id, agent_id, type);
```
This is required because `saveIntegrationAction` uses `.upsert()` with `onConflict: "business_id,agent_id,type"` — without a backing UNIQUE constraint, PostgreSQL will reject the upsert with a conflict target error.

Apply the same addition to `packages/db/schema/_combined_schema.sql` in the 014 section, after the existing `idx_integrations_agent` index.

**B) Fix agent-integrations.tsx error handling and UI refresh:**

The current `handleAddMock`, `handleSave`, and `handleDelete` functions discard the server action return value. When the action fails (e.g., table missing, constraint violation), the user sees nothing.

Add `import { toast } from "sonner"` and `import { useRouter } from "next/navigation"`.

Inside the `IntegrationTypeSection` component, add `const router = useRouter()`.

Rewrite the three handlers to follow the established pattern (see deploy-button.tsx):

```typescript
function handleSave() {
  startTransition(async () => {
    const result = await saveIntegrationAction(businessId, agentId, type, selectedProvider);
    if (result.error) {
      toast.error(result.error);
    } else {
      toast.success("Integration saved");
      router.refresh();
    }
  });
}

function handleAddMock() {
  setSelectedProvider("mock");
  startTransition(async () => {
    const result = await saveIntegrationAction(businessId, agentId, type, "mock");
    if (result.error) {
      toast.error(result.error);
    } else {
      toast.success("Mock integration added");
      router.refresh();
    }
  });
}

function handleDelete() {
  if (!integration) return;
  startTransition(async () => {
    const result = await deleteIntegrationAction(businessId, integration.id);
    if (result.error) {
      toast.error(result.error);
    } else {
      toast.success("Integration removed");
      router.refresh();
    }
  });
}
```

The `router.refresh()` call ensures the Server Component re-fetches data after the mutation, complementing the `revalidatePath` already in the server actions. The toast calls surface errors that were previously swallowed silently.
