    **packages/core/integrations/service.ts:**
    Integrations CRUD service (follows same pattern as secrets/service.ts from 03-03):

    `getIntegrationsForAgent(supabase, agentId)`:
    - Fetch all integrations where agent_id matches, ordered by type
    - Return array of integration records

    `getIntegrationsForBusiness(supabase, businessId)`:
    - Fetch all integrations where business_id matches, ordered by type then agent_id
    - Join with agents table to include agent name: `.select('*, agents(id, name)')`
    - Return array of integration records with agent name

    `upsertIntegration(supabase, businessId, agentId, type, provider, config?, status?)`:
    - Upsert an integration record for the given agent and type
    - If an integration with the same business_id + agent_id + type exists, update provider/config/status
    - Otherwise insert new record
    - Default status: 'mock' if provider starts with 'mock', otherwise 'inactive'
    - Insert audit_log with action 'integration.configured', metadata: { agent_id, type, provider }
    - Return the upserted record

    `deleteIntegration(supabase, businessId, integrationId)`:
    - Delete integration by id and business_id
    - Insert audit_log with action 'integration.deleted'

    **packages/core/index.ts:**
    Add re-exports for integrations service: getIntegrationsForAgent, getIntegrationsForBusiness, upsertIntegration, deleteIntegration

    **apps/web/_actions/integration-actions.ts:**
    "use server" module:

    `getAgentIntegrationsAction(agentId: string)`:
    - Auth check, call getIntegrationsForAgent, return { integrations } or { error }

    `getBusinessIntegrationsAction(businessId: string)`:
    - Auth check, call getIntegrationsForBusiness, return { integrations } or { error }

    `saveIntegrationAction(businessId: string, agentId: string, type: string, provider: string, config?: Record<string, unknown>)`:
    - Auth check, call upsertIntegration
    - revalidatePath for agent detail page and integrations overview
    - Return { success: true } or { error }

    `deleteIntegrationAction(businessId: string, integrationId: string)`:
    - Auth check, call deleteIntegration
    - revalidatePath for integrations pages
    - Return { success: true } or { error }

    **apps/web/_components/agent-integrations.tsx:**
    "use client" component for the Integrations tab on agent detail page:
    - Accept `agentId`, `businessId`, and `integrations` (array) as props
    - Display 5 integration type sections: CRM, Email, Helpdesk, Calendar, Messaging
    - For each type, show an IntegrationConfigCard (or inline equivalent):
      - Current provider: dropdown selector with options ["Mock", "Custom"] (for MVP, only mock is functional)
      - Status badge: mock (yellow), active (green), inactive (grey)
      - Capabilities list: call getAdapter(type).getCapabilities() client-side to show what the mock adapter supports (display as chip badges)
      - Sample data preview: show a few lines of getSampleData() output as a muted JSON snippet
      - "Save" button to persist provider selection via saveIntegrationAction
      - "Remove" button for non-mock integrations
    - If no integrations exist yet for a type, show "Not configured" with an "Add Mock" button that calls saveIntegrationAction with provider='mock'
    - Per CONTEXT.md: "Swapping mock to real: dropdown selector per integration (Mock, Salesforce, HubSpot, etc.). Selecting a real provider shows credential fields." For MVP, only Mock is functional; real provider options appear in dropdown but show a "Coming soon - configure credentials in Settings > Secrets" message when selected.
