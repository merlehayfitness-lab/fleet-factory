    **packages/core/secrets/service.ts:**
    Secrets CRUD service:

    `getSecrets(supabase, businessId)`:
    - Fetch all secrets for the business, ordered by category then key
    - Return array (encrypted_value is opaque -- the UI never sees plaintext)

    `saveSecret(supabase, businessId, key, plainValue, category, integrationType?)`:
    - Import encrypt from packages/core/crypto/encryption
    - Encrypt the plainValue
    - Upsert into secrets table (using the unique business_id + key constraint)
    - Insert audit_log with action 'secret.saved', metadata: { key, category }
    - Return the saved secret (with encrypted_value, not plain)

    `deleteSecret(supabase, businessId, secretId)`:
    - Delete the secret by id and business_id
    - Insert audit_log with action 'secret.deleted'

    `decryptSecretsForDeployment(supabase, businessId)`:
    - Fetch all secrets for the business
    - Decrypt each encrypted_value using decrypt from crypto/encryption
    - Return array of { key, decryptedValue }
    - This is called by the deployment service during artifact generation -- server-side only

    **packages/core/index.ts:**
    Add re-exports for secrets service: getSecrets, saveSecret, deleteSecret, decryptSecretsForDeployment

    **apps/web/_actions/secrets-actions.ts:**
    "use server" module:

    `getSecretsAction(businessId: string)`:
    - Auth check, call getSecrets, return { secrets } or { error }

    `saveSecretAction(businessId: string, key: string, value: string, category: string, integrationType?: string)`:
    - Auth check, call saveSecret with the plaintext value
    - revalidatePath for secrets page
    - Return { success: true } or { error }

    `deleteSecretAction(businessId: string, secretId: string)`:
    - Auth check, call deleteSecret
    - revalidatePath for secrets page
    - Return { success: true } or { error }

    **apps/web/app/(dashboard)/businesses/[id]/settings/secrets/page.tsx:**
    Server Component:
    - Extract businessId from params
    - Auth check, redirect if no user
    - Fetch business (verify exists)
    - Fetch secrets: `supabase.from('secrets').select('*').eq('business_id', businessId).order('category').order('key')`
    - Page heading: "Secrets & Credentials" with link back to deployment center
    - Render SecretsManager component with secrets data and businessId

    **apps/web/_components/secrets-manager.tsx:**
    "use client" component for managing secrets (Vercel env vars style per CONTEXT.md):
    - Accept `secrets` array and `businessId` as props
    - **Add Secret Form:**
      - Key input (text, uppercase convention hint)
      - Value input (type="password" by default)
      - Category select: API Key, Credential, Token
      - Optional integration type select: CRM, Email, Helpdesk, Calendar, Messaging, or blank
      - "Add Secret" button
      - On submit: call saveSecretAction, show toast, clear form on success
    - **Secrets List:**
      - Group secrets by category (API Keys, Credentials, Tokens)
      - Each category as a section with heading
      - Each secret row shows:
        - Key name (bold)
        - Masked value: "**********************" by default
        - Eye icon button to reveal -- when clicked, shows the encrypted_value (which is still encrypted, but shows it's stored). For MVP, we show "Encrypted - value hidden" since decryption only happens server-side during deployment.
        - Integration type badge (if set)

      - Per CONTEXT.md: masked by default, reveal on click (for MVP reveal shows "Value stored securely" since we cannot decrypt client-side)
    - If no secrets, show empty state: "No secrets configured. Add API keys and credentials for your integrations."

    IMPORTANT: Secrets are NEVER decrypted on the client side. The reveal button shows a confirmation that the value is stored, not the actual plaintext. Decryption only happens server-side during deployment artifact generation.
