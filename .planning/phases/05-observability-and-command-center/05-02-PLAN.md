    **packages/core/emergency/emergency-service.ts:**
    Emergency action service. Accept SupabaseClient as first arg. Every function validates, executes the change, and creates an audit_log entry with the reason.

    ```typescript
    // freezeAgentWithReason(supabase, agentId, businessId, reason: string, actorId: string)
    // 1. Fetch agent to verify it exists and get current status
    // 2. Call assertTransition to validate the freeze transition
    // 3. Update agent status to 'frozen'
    // 4. Create audit_log with action='emergency.agent_frozen', metadata includes { reason, previous_status, agent_name, actor_id }
    // Return void (throws on error)

    // revokeToolAccess(supabase, agentId, businessId, reason: string, actorId: string)
    // 1. Fetch agent to verify
    // 2. Store the current tool_profile in a backup field in metadata (for potential restore)
    // 3. Update agent's tool_profile to {} (empty -- no tools)
    // 4. Create audit_log with action='emergency.tools_revoked', metadata includes { reason, previous_tool_count, agent_name }

    // disableAgent(supabase, agentId, businessId, reason: string, actorId: string)
    // 1. Fetch agent to verify
    // 2. Transition to 'retired' status via assertTransition
    // 3. Create audit_log with action='emergency.agent_disabled', metadata includes { reason, previous_status, agent_name }

    // restoreAgent(supabase, agentId, businessId, actorId: string)
    // 1. Fetch agent -- must be in 'frozen' status
    // 2. Transition from frozen -> active via assertTransition
    // 3. Create audit_log with action='emergency.agent_restored', metadata includes { agent_name }

    // disableTenant(supabase, businessId, reason: string, actorId: string)
    // 1. Fetch business to verify it exists and get current status
    // 2. Update business status to 'disabled'
    // 3. Fetch all non-retired agents for this business
    // 4. For each agent that can be frozen (active, paused, error), transition to 'frozen'
    // 5. Create audit_log with action='emergency.tenant_disabled', metadata includes { reason, agents_frozen_count }

    // restoreTenant(supabase, businessId, actorId: string)
    // 1. Fetch business -- must be in 'disabled' status
    // 2. Update business status to 'active'
    // 3. Do NOT auto-unfreeze agents -- admin must manually review and restore each agent
    // 4. Create audit_log with action='emergency.tenant_restored', metadata includes { note: "Agents remain frozen for manual review" }
    ```

    All audit logging is best-effort (errors logged not thrown).

    **apps/web/_actions/log-actions.ts:**
    "use server" module:

    `getAuditLogs(businessId, filters?: { actor?: string, eventType?: string, dateFrom?: string, dateTo?: string, entityType?: string, search?: string, limit?: number, offset?: number })`:
    - Auth check
    - Build Supabase query on audit_logs with:
      - business_id filter (always)
      - Optional filters: action match for eventType, entity_type match, created_at range for dates
      - Free-text search: `metadata::text ILIKE '%search%'` (Option B from research) OR `action ILIKE '%search%'`
      - Pagination via limit/offset
      - Order by created_at DESC
    - Return { logs, totalCount } or { error }

    `exportAuditLogs(businessId, filters, format: 'csv' | 'json')`:
    - Auth check
    - Fetch all matching logs (no pagination limit)
    - Return { data: logs } -- actual CSV/JSON conversion happens client-side
    - Return { data } or { error }
