    Chat CRUD service. Accept SupabaseClient as first arg.

    ```typescript
    // getOrCreateConversation(supabase, businessId, departmentId, userId)
    // 1. Query conversations for this business + department + user where status='active'
    // 2. If found, return existing conversation
    // 3. If not found, INSERT new conversation with title=null, status='active'
    // 4. Return the conversation (id, businessId, departmentId, etc.)
    // NOTE: This creates one conversation per department per user

    // sendMessage(supabase, businessId, conversationId, content, role, agentId?, toolCalls?, metadata?)
    // 1. INSERT message with provided fields
    // 2. UPDATE conversation: increment message_count, set last_message_at = now(), updated_at = now()
    // 3. Return the created message (with id and created_at)

    // getMessages(supabase, conversationId, limit = 50, before?: string)
    // 1. Query messages for this conversation ordered by created_at ASC
    // 2. If 'before' provided, filter where created_at < before (for "load more" pagination)
    // 3. Join with agents table to get agent_name for agent messages
    // 4. Limit to the specified count
    // 5. Return array of ChatMessage objects

    // getConversationsForBusiness(supabase, businessId)
    // 1. Query conversations where business_id matches, ordered by last_message_at DESC
    // 2. Join with departments for department_name and department_type
    // 3. Return array of ChatConversation objects

    // archiveConversation(supabase, conversationId)
    // 1. UPDATE conversation status to 'archived', set updated_at
    // 2. Create audit_log with action='conversation.archived'

    // getDepartmentChannels(supabase, businessId, userId)
    // 1. Query all departments for this business
    // 2. For each department, check if an active conversation exists (for this user)
    // 3. Check if the department has an active agent (for hasActiveAgent/agentFrozen indicators)
    // 4. Count messages after the user's last message as unread (simple heuristic)
    // 5. Return array of DepartmentChannel objects

    // routeAndRespond(supabase, businessId, departmentId, conversationId, userMessage)
    // 1. Use selectAgent from orchestrator/router.ts to find the right agent
    // 2. If no active agent found, check if agent is frozen -- if so, return a system message "Agent is frozen"
    // 3. If active agent found, call generateStubResponse(departmentType, userMessage)
    // 4. Create agent message via sendMessage with the stub response content, tool_calls, and agent_id
    // 5. Create audit_log with action='chat.message_routed' and metadata { agentId, departmentId }
    // 6. Return the agent message
    ```
