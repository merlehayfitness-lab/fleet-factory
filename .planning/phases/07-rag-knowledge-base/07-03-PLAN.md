    Modify `routeAndRespond()` to inject RAG context:

    Find the point in routeAndRespond where the agent has been selected (after selectAgent call) but before the message is sent to VPS or stub. Add knowledge retrieval:

    ```typescript
    // --- RAG Knowledge Injection ---
    // Retrieve relevant knowledge context before routing to agent
    let knowledgeContext: RetrievedContext | null = null;
    try {
      const { retrieveKnowledgeContext } = await import("../knowledge/retriever");
      knowledgeContext = await retrieveKnowledgeContext(
        supabase,
        businessId,
        agent.id,    // agent ID for two-tier scoping
        userMessage,  // the user's message as the query
        { conversationId }
      );
    } catch (err) {
      // Best-effort: if retrieval fails (e.g., no OPENAI_API_KEY), continue without context
      console.error("Knowledge retrieval failed, proceeding without context:", err);
    }
    ```

    For the VPS path (sendChatToVps call):
    - Include knowledgeContext.contextString in the VpsChatRequest payload as the knowledgeContext field
    - If knowledgeContext is null or contextString is empty, omit the field

    For the stub path (generateStubResponse call):
    - Knowledge context is cosmetic for stubs, but include sources in agent message metadata

    After creating the agent message (both paths):
    - If knowledgeContext has sources, include them in the agent message metadata:
      ```typescript
      metadata: {
        ...existingMetadata,
        knowledgeSources: knowledgeContext?.sources ?? [],
      }
      ```

    Modify `executeTask()` to inject RAG context:

    Find the point in executeTask where the task is about to be sent to VPS or mock execution. Add knowledge retrieval:

    ```typescript
    // --- RAG Knowledge Injection ---
    let knowledgeContext: RetrievedContext | null = null;

    } catch (err) {

    }
    ```
