    //   - Auth check via supabase.auth.getUser(), redirect if not authenticated

    //   - Return { document } immediately -- DO NOT call processDocument() here
    //   - The client is responsible for calling triggerProcessingAction after this returns
    //   - IMPORTANT: Do NOT wrap redirect() in try/catch

    // triggerProcessingAction(documentId: string)
    //   - Auth check
    //   - Fetch the document record to get storage_path, file_type, business_id
    //   - Download file buffer from Supabase Storage using storage_path
    //   - Call processDocument(supabase, documentId, fileBuffer, fileType)
    //     This is the long-running operation (extract + chunk + embed) that
    //     transitions status from uploading -> processing -> ready/failed
    //   - Return { success: true, chunkCount } or { error }
    //   - The client calls this as fire-and-forget (no await), then polls
    //     via listDocumentsAction to observe status transitions

    // pasteTextAction(businessId, agentId, title, content)
    //   - Auth check
    //   - Validate title is not empty and content is not empty
    //   - Call createDocument() with fileType='text', filename=null, fileSizeBytes=content.length, status='uploading'
    //   - Return { document } immediately -- DO NOT call processDocument() here
    //   - The client calls triggerProcessingAction after this returns

    // deleteDocumentAction(documentId)
    //   - Auth check
    //   - Call deleteDocument() from knowledge service
    //   - revalidatePath

    //   - Auth check
    //   - Call reIndexDocument() from knowledge service

    //   - Auth check

    **apps/web/_components/knowledge-upload-zone.tsx:**
    "use client" drag-and-drop upload zone:

    Props:
    - `businessId: string`
    - `agentId: string | null` (null for global scope)
    - `onUploadComplete?: () => void` (callback to refresh document list)

    Layout:
    - Dashed border drop zone area with "Drag & drop files here" text and file icon
    - Supported file types shown below: ".txt, .md, .pdf, .docx, .xlsx (max 25MB)"
    - Hidden file input triggered by clicking the zone, accepts multiple files
    - Below the drop zone: a collapsible "Or paste text" section
      - Text area for pasting content
      - Title input field (required for pasted text)
      - "Add Text" button
    - Upload progress: show file names with spinner during upload

    Behavior:
    - On file drop/select: for each file:
      1. Call uploadDocumentAction with FormData -- this returns immediately with { document }
      2. Call onUploadComplete callback to trigger list refresh (document appears with 'uploading' status)
      3. Fire triggerProcessingAction(document.id) as fire-and-forget (no await) -- this runs
         the long processing pipeline server-side while the UI remains responsive
      4. The KnowledgeDocumentList polling picks up status transitions automatically
    - On paste text submit: same two-phase pattern:
      1. Call pasteTextAction -- returns immediately with { document }
      2. Call onUploadComplete, then fire triggerProcessingAction(document.id) without awaiting
    - Show toast on upload success ("Document uploaded -- processing started") or error
    - Validate file type and size client-side before submitting (show toast error for invalid files)

    DnD implementation:
    - onDragOver: prevent default, set isDragging state
    - onDragLeave: unset isDragging
    - onDrop: prevent default, extract files from event.dataTransfer, process each
    - isDragging state: highlight border with accent color
