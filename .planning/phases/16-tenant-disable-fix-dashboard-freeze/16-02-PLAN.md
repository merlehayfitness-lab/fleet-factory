  <action>
    Add `import { requireActiveBusiness } from "@/_lib/require-active-business";` to each action file that has mutation functions.

    For each mutation function, add the guard call AFTER auth check but BEFORE any write operations:
    ```typescript
    const guard = await requireActiveBusiness(businessId);
    if (guard) return guard;
    ```

    **IMPORTANT EXEMPTIONS** -- do NOT add the guard to any function in `emergency-actions.ts`. That file is NOT listed in this task's files. Emergency actions must work regardless of business status. The disableTenant, restoreTenant, freezeAgent, restoreAgent, revokeTools, and disableAgent functions are all emergency operations.

    Also do NOT add the guard to:
    - All read-only/getter actions across all files (getDeploymentsAction, getDeploymentStatusAction, getMessages, getTasks, getApprovals, etc.)
    - `health-actions.ts` (read-only)
    - `log-actions.ts` (read-only)
    - `auth-actions.ts` (no business scope)

    **Specific files and their mutation functions to guard:**

    `deployment-actions.ts`:
    - `deployAction(businessId)` -- add guard
    - `retryDeploymentAction(businessId, deploymentId)` -- add guard
    - `rollbackDeploymentAction(businessId, targetVersion)` -- add guard
    - `deployAgentAction(businessId, agentId)` -- add guard
    - `getDeploymentsAction` -- NO guard (read-only)
    - `getDeploymentStatusAction` -- NO guard (read-only)

    `agent-actions.ts`:
    - All mutation functions (update, delete, config changes, reparent) -- add guard
    - All getter/read functions -- NO guard

    `agent-wizard-actions.ts`:
    - All mutation functions (create provisional, update wizard, finalize) -- add guard

    `approval-actions.ts`:
    - `approveAction`, `rejectAction`, `bulkApprove`, `bulkReject` -- add guard
    - `getApprovals` -- NO guard

    `chat-actions.ts`:
    - `sendMessageAction` -- add guard
    - Read functions -- NO guard

    `task-actions.ts`:
    - `createTaskAction`, `updateTaskAction`, `deleteTaskAction` -- add guard
    - Read functions -- NO guard

    `integration-actions.ts`:
    - All create/update/delete mutations -- add guard
    - Read functions -- NO guard

    `knowledge-actions.ts`:
    - Upload, delete, triggerProcessing mutations -- add guard
    - Read functions -- NO guard

    `prompt-generator-actions.ts`:
    - Generate and refine mutations -- add guard

    `secrets-actions.ts`:
    - Save, delete, saveProviderCredentials, deleteProviderSecrets -- add guard
    - Read/reveal -- NO guard (reading secrets when disabled is allowed)

    `skill-actions.ts`:
    - Create, update, delete, assign, unassign -- add guard
    - Read functions -- NO guard

    `template-actions.ts`:
    - Create, update mutations -- add guard
    - Read functions -- NO guard

    `vps-actions.ts`:
    - Any mutation functions -- add guard
    - Health check reads -- NO guard

    `business-actions.ts`:
    - `updateBusiness` -- add guard
    - `createBusiness` -- NO guard (creating a new business is always allowed)
    - Read functions -- NO guard

    For functions that do not directly receive `businessId` as a parameter but derive it from other data, use the appropriate ID extraction pattern. If the function only has agentId, look up the businessId first OR skip the guard if the function is only called from contexts where the layout already enforces the check.

    **Pattern for adding the guard:**
    After the auth check (after `if (!user) { redirect("/sign-in"); }`), add:
    ```typescript
    const guard = await requireActiveBusiness(businessId);
    if (guard) return guard;
    ```

    This is a mechanical change. Each mutation function gets 2 new lines. The return type already supports `{ error: string }` in all action files.
