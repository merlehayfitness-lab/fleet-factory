<objective>
Create the 5-step agent setup wizard with knowledge upload and prompt generation steps.

Purpose: This plan adds the ability to create new agents for an existing business through a guided wizard. The wizard flow is: Basic Info (name, department) -> Knowledge Upload (agent-specific docs, blocks until processed) -> Role Definition (description, tone, focus with templates) -> Prompt Generation (Claude generates, admin refines) -> Review (full summary with Create Agent button). This fills the gap of having no standalone "add agent" flow -- currently agents are only created through business provisioning.

Output: 1 wizard component, 1 server action file, 1 page route, 1 modified agents list page.
</objective>

    **apps/web/_components/agent-setup-wizard.tsx:**
    "use client" component with 5-step wizard. Follow the same useState step-navigation pattern as create-business-wizard.tsx.

    Props:
    - businessId, businessName, businessIndustry: string
    - departments: Array<{ id: string; name: string; type: string }>
    - templates: Array<{ id: string; name: string; department_type: string; system_prompt: string; tool_profile: Record<string, unknown>; model_profile: Record<string, unknown> }>

    Step indicators at top showing current step (1-5) with labels.

    **Step 1: Basic Info**
    - Agent name input (required)
    - Department selector dropdown (from departments prop)
    - Optional: template selector filtered by selected department's type
    - If template selected, pre-fill system_prompt, tool_profile, model_profile from template
    - Next button enabled when name and department are selected

    **Step 2: Knowledge Upload**
    - **Provisional agent creation:** When the user arrives at Step 2 for the first time, automatically create a provisional agent record via a new `createProvisionalAgentAction(businessId, name, departmentId)` server action. This inserts into the agents table with `status: 'provisioning'`, the name and department_id from Step 1, and empty/default values for other fields. Store the returned `agentId` in wizard state. This ensures knowledge uploads have a real agent_id foreign key.
    - If the user goes back to Step 1 and changes name/department, update the provisional record via `updateProvisionalAgentAction`.
    - Reuse KnowledgeUploadZone component (from Phase 7) with the provisional `agentId`.
    - Per CONTEXT: "Block until ready" -- show processing progress per document. Next button disabled while any document is in uploading/processing state. All documents must reach 'ready' status before the admin can proceed.
    - Per CONTEXT: "Wizard only handles agent-specific uploads, not global business-wide docs"
    - No skip option. Knowledge upload is a required stop (per locked decision), though uploading zero documents is allowed -- the blocking requirement applies to in-progress uploads, not to having zero docs.
    - "Next" button only. Enabled when no documents are currently uploading/processing (zero docs = enabled, all docs ready = enabled, any doc in-flight = disabled).

    **Step 3: Role Definition**
    - Description textarea, tone input, focus input (same fields as RoleDefinitionCard)
    - Template selector dropdown (loads ROLE_TEMPLATES for the selected department type from Step 1)
    - Selecting a role template populates the 3 fields
    - Fields are optional -- admin can skip role definition and write prompt manually
    - Next button always enabled (fields are optional)

    **Step 4: Prompt Generation**
    - If role definition fields are filled (Step 3), show "Generate System Prompt" button
    - Clicking it runs the two-step flow: parseRoleDefinitionAction -> show breakdown -> generatePromptAction
    - Context suggestions: show knowledge doc titles (from Step 2) and integration types as checkboxes
    - After generation, show PromptGenerationPanel for refinement
    - If role definition was skipped, show a plain textarea for manual system prompt entry
    - "Accept" button saves the prompt to wizard state
    - If template was selected in Step 1, pre-fill prompt from template and allow editing

    **Step 5: Review**
    - Full summary showing:
      - Agent name, department
      - Role definition summary (description, tone, focus) if provided
      - Generated system prompt (collapsible per section: Identity, Instructions, Tools, Constraints)
      - Knowledge documents list with status badges (fetched for the provisional agentId)
      - Template reference if selected
    - "Create Agent" button calls finalizeAgentAction with the provisional agentId and all collected data (system_prompt, role definition fields, tool_profile, model_profile, templateId)
    - On success, redirect to /businesses/[id]/agents/[agentId] (same agentId, now active)
    - On error, show error toast and stay on review step
