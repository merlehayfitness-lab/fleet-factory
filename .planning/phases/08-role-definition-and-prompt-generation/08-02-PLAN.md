    Add two new functions:
    - `getChildAgents(supabase, parentAgentId, businessId)` -- queries agents where parent_agent_id matches and business_id matches. Returns array of {id, name, status, role}.
    - `getParentAgent(supabase, agentId, businessId)` -- fetches the agent, checks parent_agent_id, if set fetches the parent agent. Returns {id, name, status, role} or null.

    Also extend `updateAgentConfig` to accept `role` (string) and `parent_agent_id` (string | null) in the config parameter, with corresponding entries in the update payload builder.

    **packages/runtime/generators/openclaw-skill-md.ts:**
    New pure function SKILL.md generator:
    - `generateSkillMd(agentName: string, departmentSkill: string | null, agentSkill: string | null): string`
    - If both are null/empty, return a minimal default SKILL.md with just the agent name header and a "No specific skills defined" note
    - If only departmentSkill exists, use it as the base SKILL.md content
    - If only agentSkill exists, use it directly
    - If both exist, merge: department skill provides the baseline sections, agent skill content overrides/appends. Simple strategy: concatenate with department skill first under "## Department Baseline" and agent skill under "## Agent Specialization", wrapped in a proper SKILL.md structure with frontmatter (name, description)
    - Character budget: ~4000 chars max, truncate if exceeded

    Extend AgentInput interface with `skill_definition: string | null`.
    Extend DepartmentInput interface with `department_skill: string | null`.
    In the per-agent file generation loop, after generating USER.md, add SKILL.md generation:
    ```typescript
    files.push({
      path: `${workspaceDir}/SKILL.md`,
      content: generateSkillMd(agent.name, dept.department_skill, agent.skill_definition),

    ```
    Import generateSkillMd from `./openclaw-skill-md`.
