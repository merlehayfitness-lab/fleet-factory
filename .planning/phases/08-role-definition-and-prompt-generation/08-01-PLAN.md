    **packages/core/prompt-generator/generator-types.ts:**
    Type definitions for the prompt generation system:
    - `RoleDefinition` interface: `description` (string), `tone` (string), `focus_areas` (string[]), `workflow_instructions` (string), `linked_integrations` (string[]), `linked_knowledge_docs` (string[]), `template_id` (optional string)
    - `PromptSections` interface: `identity` (string), `instructions` (string), `tools` (string), `constraints` (string)
    - `GenerationResult` interface: `promptSections` (PromptSections), `skillDefinition` (string), `structuredBreakdown` (string -- human-readable summary of what Claude understood)
    - `RefinementRequest` interface: `currentSections` (PromptSections), `refinementMessage` (string), `conversationHistory` (array of {role, content})
    - `RefinementResult` interface: `updatedSections` (PromptSections), `changeDescription` (string)
    - `TestChatMessage` interface: `role` ("user" | "assistant"), `content` (string)

    **packages/core/prompt-generator/prompt-templates.ts:**
    Meta-prompts that instruct Claude how to generate agent prompts and SKILL.md:
    - `buildGenerationSystemPrompt()` -- returns a system prompt that instructs Claude to:
      1. Analyze the role definition inputs (description, tone, focus, workflow)
      2. Output a JSON object with fields: `identity`, `instructions`, `tools`, `constraints`, `skillDefinition`, `structuredBreakdown`
      3. Identity section: who the agent is, its role within the department
      4. Instructions section: step-by-step workflow derived from admin's free-text workflow instructions
      5. Tools section: which tools to use and when, based on linked integrations
      6. Constraints section: boundaries, tone enforcement, forbidden actions
      7. skillDefinition: markdown SKILL.md with Capabilities, Tools, Task Boundaries, Workflows sections
      8. structuredBreakdown: plain-language summary of parsed steps for admin confirmation
      Include character budgets: system prompt ~4000 chars total, SKILL.md ~4000 chars
    - `buildGenerationUserMessage(roleDefinition, businessContext, knowledgeDocTitles, integrationNames)` -- formats the role definition inputs and context into the user message
    - `buildRefinementSystemPrompt()` -- instructs Claude to refine an existing prompt based on a request, outputting updated sections as JSON
    - `buildRefinementUserMessage(currentSections, refinementMessage, conversationHistory)` -- formats current prompt + refinement request

    **packages/core/prompt-generator/generator-service.ts:**
    Main generation orchestrator:
    - `getAnthropicApiKey()` helper -- throws if ANTHROPIC_API_KEY missing, following the pattern from `packages/core/knowledge/embedder.ts`
    - `getAnthropicClient()` factory -- creates `new Anthropic({ apiKey })` instance
    - `generatePromptAndSkill(roleDefinition, businessContext, knowledgeDocTitles, integrationNames)` -- calls Claude with meta-prompt, parses JSON response into GenerationResult. Use `claude-sonnet-4-20250514` model. Wrap in try/catch with descriptive errors. businessContext is `{ name, industry, departmentType }`.

    **packages/core/prompt-generator/refinement-service.ts:**
    Iterative refinement:
    - `refinePrompt(request: RefinementRequest)` -- calls Claude with refinement system prompt + current sections + refinement message + conversation history. Returns RefinementResult with updated sections and change description. Uses same Anthropic client and model.

    **packages/core/prompt-generator/test-chat-service.ts:**
    Test chat with draft prompts:
    - `sendTestMessage(systemPrompt: string, messages: TestChatMessage[], modelProfile?: Record<string, unknown>)` -- calls Claude with the draft system prompt as the system message, messages as conversation history. Returns assistant response string. Model defaults to claude-sonnet-4-20250514 unless modelProfile specifies otherwise.
