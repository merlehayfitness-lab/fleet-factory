    **apps/web/_actions/agent-actions.ts:**
    Add two new Server Actions:

    1. `getTemplateDiffAction(agentId: string, businessId: string)`:
       - Auth check (createServerClient, getUser, redirect if no user)
       - Fetch agent: SELECT id, template_id, tool_profile, model_profile FROM agents WHERE id AND business_id
       - If no template_id, return `{ error: "Agent has no linked template" }`
       - Fetch template: SELECT tool_profile, model_profile FROM agent_templates WHERE id = agent.template_id
       - Return `{ agent: { tool_profile, model_profile }, template: { tool_profile, model_profile }, hasChanges: boolean }`
       - hasChanges: JSON.stringify comparison of both profiles

    2. `syncFromTemplateAction(agentId: string, businessId: string)`:
       - Auth check
       - Import `syncFromTemplate` from `@fleet-factory/core/server` (or from the core server barrel)
       - Call `syncFromTemplate(supabase, agentId, businessId)`
       - Revalidate path: `/businesses/${businessId}/agents/${agentId}`
       - Return `{ success: true, before, after }` or `{ error: string }`

    **apps/web/_components/sync-from-template-dialog.tsx:**
    "use client" controlled Dialog for sync confirmation with diff preview.
    Props: `agentId` (string), `businessId` (string), `hasTemplate` (boolean -- whether agent has a linked template).

    Layout:
    - Trigger button: "Sync from Template" with refresh icon. Disabled if `!hasTemplate`.
    - On click (before opening dialog): call `getTemplateDiffAction` to fetch the diff data
    - Loading state while fetching diff

    Dialog content:
    - Title: "Sync from Template"
    - Description: "This will overwrite the agent's model and tool profiles with the template's current values."

    - If `hasChanges` is false: show "Profiles are already in sync with the template. No changes needed." with only a Close button.

    - If `hasChanges` is true: show diff sections:
      - **Model** section (only if model differs):
        - Left: "Current: {agent model friendly name}" with muted styling
        - Right: "Template: {template model friendly name}" with highlighted styling
        - Use `getModelFriendlyName` for display
      - **Tool Profile** section (only if tool profile differs):
        - Left: "Current" with JSON preview (max-h-32 overflow)
        - Right: "Template" with JSON preview (highlighted border)
      - Footer: "Cancel" and "Overwrite Agent Profiles" buttons

    - On confirm: call `syncFromTemplateAction`, show toast on success "Profiles synced from template", show redeploy toast: `toast("Profiles synced", { action: { label: "Redeploy", onClick: () => ... } })`
    - On error: show toast.error with error message
    - Saving state with disabled confirm button
