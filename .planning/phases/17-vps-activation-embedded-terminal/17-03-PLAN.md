    **infra/vps/SETUP-CHECKLIST.md** (NEW):
    Comprehensive manual setup and verification guide for activating the VPS.

    Structure:

    # VPS Activation Checklist

    ## Prerequisites
    - [ ] Hostinger VPS with Ubuntu 22+ or Debian 12+
    - [ ] Root SSH access to the VPS
    - [ ] Agency Factory admin app deployed to Vercel with VPS_API_URL and VPS_API_KEY env vars
    - [ ] Anthropic API key for Claude API access on VPS
    - [ ] OpenClaw account and auth token

    ## Step 1: Run Setup Script
    ```bash
    # SSH into VPS
    ssh root@your-vps-ip

    # Upload and run setup script
    scp infra/vps/* root@your-vps-ip:/tmp/agency-factory-setup/
    cd /tmp/agency-factory-setup
    chmod +x setup.sh
    ./setup.sh
    ```

    This installs:
    - Node.js 24
    - Docker
    - OpenClaw CLI
    - Creates /data/tenants/ and /data/state/ directories
    - Copies API proxy files to /opt/agency-factory/vps-proxy/
    - Installs npm dependencies (express, ws, ssh2, dockerode, dotenv)
    - Creates systemd service

    ## Step 2: Configure Environment
    ```bash
    nano /opt/agency-factory/vps-proxy/.env
    ```

    Required values:
    - `API_KEY` = shared secret (must match VPS_API_KEY in Vercel)
    - `OPENCLAW_AUTH_TOKEN` = your OpenClaw gateway token
    - `OPENCLAW_HTTP_URL` = http://127.0.0.1:18789
    - `OPENCLAW_WS_URL` = ws://127.0.0.1:18789
    - `TENANT_DATA_DIR` = /data/tenants
    - `STATE_DIR` = /data/state
    - `SSH_USER` = root (or another user with Docker access)
    - `SSH_PASSWORD` = your SSH password (for terminal bridge)

    ## Step 3: Configure OpenClaw
    ```bash
    openclaw config set api.http.enabled true
    openclaw config set api.http.port 18789
    openclaw daemon restart
    ```

    Verify:
    ```bash
    curl http://127.0.0.1:18789/api/status
    # Should return JSON with status
    ```

    ## Step 4: Pull Docker Sandbox Image
    ```bash
    docker pull openclaw-sandbox-common:bookworm-slim
    ```

    If the image is not available from a registry, build locally:
    ```bash
    # Check OpenClaw docs for sandbox image build instructions
    openclaw sandbox build
    ```

    ## Step 5: Configure SSH for Terminal Bridge
    The embedded terminal connects via SSH to localhost. Ensure SSH daemon is running:
    ```bash
    systemctl status sshd
    # If not running:
    apt-get install -y openssh-server
    systemctl enable sshd
    systemctl start sshd
    ```

    Verify the SSH credentials in .env match a user that can log in:
    ```bash
    ssh root@127.0.0.1
    # Should succeed with the password from .env
    ```

    ## Step 6: Start the API Proxy
    ```bash
    systemctl start agency-factory-proxy
    systemctl status agency-factory-proxy
    ```

    Verify:
    ```bash
    curl http://localhost:3100/healthz
    # Should return { "ok": true, "timestamp": "..." }

    curl -H "X-API-Key: YOUR_API_KEY" http://localhost:3100/api/health
    # Should return { "status": "online"|"degraded", "agentCount": 0, ... }
    ```

    ## Step 7: Bootstrap Claude Code
    ```bash
    openclaw chat < /opt/agency-factory/vps-proxy/bootstrap-prompt.md
    ```

    This gives Claude Code its operational context for managing agent workspaces.

    ## Step 8: Configure Admin App
    In Vercel (or .env.local for development):
    ```
    VPS_API_URL=https://your-vps-ip:3100
    VPS_API_KEY=your-shared-api-key
    ```

    Verify from admin app: the VPS status indicator on business dashboard should show "Online" or "Degraded" (not "Not configured").

    ## End-to-End Verification Scenario

    Follow this sequence to verify the full loop works:

    ### 1. Create a Business
    - Go to /businesses/new in admin panel
    - Create "Test Corp" with industry "Technology"
    - Verify 4 default departments and starter agents are created

    ### 2. Deploy to VPS
    - Go to /businesses/{id}/deployments
    - Click "Deploy"
    - Watch deployment progress stream (should show real phases: writing files, optimizing, starting containers)
    - Verify deployment status reaches "Live"
    - Check VPS terminal: `docker ps` should show running containers for each agent

    ### 3. Chat with an Agent
    - Go to /businesses/{id}/chat
    - Select the Sales department
    - Send: "What services does Test Corp offer?"
    - Verify agent responds with a real Claude response (not stub text)

    ### 4. Create a Task
    - Go to /businesses/{id}/tasks
    - Create task: "Qualify lead: John Smith at Acme Inc, interested in AI consulting"
    - Assign to Sales department
    - Verify task routes to VPS agent and executes

    ### 5. Verify Approval Flow
    - Create a high-priority task that triggers approval
    - Go to /businesses/{id}/approvals
    - Verify approval request appears
    - Approve it
    - Verify result flows back from VPS

    ### 6. Test Terminal Access
    - On business dashboard, click the Terminal icon next to VPS status
    - Verify terminal opens with dark theme
    - Run: `ls` -- should show tenant workspace files
    - Run: `docker ps` -- should show running agent containers
    - Run: `cat workspaces/*/AGENTS.md` -- should show agent config

    ### 7. Verify Health Monitoring
    - Dashboard VPS status should show "Online"
    - Agent health grid should show running agents
    - Stop a container: `docker stop {agent-name}`
    - Wait 30s -- agent health should update to show stopped state
    - Restart: `docker start {agent-name}`
    - Wait 30s -- agent health should recover

    ## Troubleshooting

    ### VPS Status Shows "Offline"
    - Check proxy is running: `systemctl status agency-factory-proxy`
    - Check port is open: `curl http://localhost:3100/healthz`
    - Check API key matches between admin app and VPS .env
    - Check firewall allows port 3100 from Vercel IP

    ### Deployment Fails at "Optimizing"
    - Check OpenClaw gateway is running: `curl http://127.0.0.1:18789/api/status`
    - Check OPENCLAW_AUTH_TOKEN is valid
    - Check Claude Code was bootstrapped: `openclaw sessions list`

    ### Agent Not Responding to Chat
    - Check container is running: `docker ps | grep {business-slug}`
    - Check OpenClaw gateway routes to agent: `curl -H "Authorization: Bearer TOKEN" http://127.0.0.1:18789/api/sessions/{agent-id}/messages`
    - Check agent workspace files exist: `ls /data/tenants/{slug}/workspaces/`

    ### Terminal Connection Fails
    - Check SSH daemon: `systemctl status sshd`
    - Check SSH credentials in .env match a valid user
    - Check proxy logs: `journalctl -u agency-factory-proxy -f` and look for "[ws:terminal]" entries
    - Verify the WebSocket URL includes the correct apiKey parameter

    ### Docker Permission Issues
    - Ensure proxy runs as root or a user in the docker group
    - Check socket permissions: `ls -la /var/run/docker.sock`
